Esta sección se describe el proceso realizado para lograr la generación de los archivos RDF a través de un proceso ETL utilizando la herramienta GeoKettle. Para ello, se configura una transformación para que la generación de los archivos rdf se realice de una manera simple utilizando algunos plugins existentes en la herramientas y también el plugin Geometry2RDF.

Para la generacion de los links SameAs se utiliza la herramienta Silk Workbench. Además, hemos desarrollado plugins que permiten descubrir links entre diferentes recursos aplicando una serie de HEURISTICAS que se describen en la seccion "Generacion de Links SameAS"

La generacion de los archivos RDF la realizamos a traves de un proceso ETL. Para ello hemos desarrollado un plugin basado en la libreria Geometry2RDF para la herramienta Geokettle que permite manipular información en el entorno ETL. Se configura una transformación para que la generación de los archivos rdf se realice de forma simple utilizando Geometry2RDF y otros plugins existentes en GeoKettle

Descripcion de cada unos de los plugins utilizados

Este plugin permite seleccionar el archivo ShapeFile (*.shp) para leer la informacion que contiene el mencionado archivo *.shp.
Este plugin permite transformar el sistema de coordenadas, para ello el usuario indica el sistema de coordenadas de la fuente (En este caso de la geometria del archivo *.shp) o el plugin puede detectar en forma Automatica. Posteriormente el usuario selecciona el sistema de coordenadas al que desea transformar la geometria

Geometry2RDF es un plugin que ha sido desarrollado basado en Geometry2RDF library [Fuente], este plugin permite generar RDF desde una entrada con informacion Geometrica, por ejemplo: ShapeFile, PostGis, GDB, MySql Spatial, Oracle Spatial y cualquier fuente de informacion que pueda ser soportada por GeoKettle.

Para configurar el plugin el usuario debe indicar el nombre de la columna o propiedad que servira para identificar a cada uno de los recursos, al generar el archivo RDF los datos de esta columna se convertiran en el dato de la propiedad LABEL. Posteriormente el usuario debe espeficicar el nombre de la clase que identifique a todos los recursos contenidos en el archivo RDF que sera generado. El campo "Shapefile-path" puede quedar en blanco, este campo se utiliza para generar RDF de los Metadatos del ShapeFile, para este proceso se ha creado otro video tutorial.
Este plugin permite indicar la ubicacion del archivo de salida, formato del archivo ( RDF: *.rdf ), codificacion (UTF-8), ademas, se debe indicar el nombre del campo proveniente de Geometry2RDF que contiene la informacion en formato RDF

Para encontrar o descubrir enlaces entre recursos que se encuentran en diferentes fuentes de informacion pero que hacen referencia al mismo objeto del mundo real se utiliza la herramienta Silk Workbench.
Esta herramienta permite encontrar duplicados de los recursos, en base a condiciones que el usuario puede especificar y decidir con que propiedades desea trabajar.Generalmente para encontrar los duplicados se puede hacer referencia a las etiquetas de los recursos ( propiedad "Label" ), donde se compara los String de cada una de las etiquetas de los recursos de la fuente A con las etiquetas de los recursos de la fuente B.
Basado en Heuristicas para descubrir duplicados de recursos hemos desarrollado plugins que permiten trabajar con la información geometrica de cada recurso. En este caso se realiza comparaciones segun la geometría, área o perímetro para asi encontrar duplicados apegados a un margen de error dado por el usuario

Sin importar el tipo de geometría (Puntos, Curvas o Polígonos), los plugins desarrollados obtienen la informacion del recurso que debe estar en el formato WKT para luego realizar las respectivas comparaciones y devolver como resultado links SameAS

Descripcion de los Plugis Desarrollados

Este plugin permite encontrar relaciones sameAs entre diferentes recursos que esten relacionados con su geometría. En este caso , utilizando GEOTOOLS, se aplica la función buffer que permite saber si la geometría A contiene a la geometria B, dando como resultado la posibilidad de que se trate del mismo recurso del mundo real. El valor de buffer es ingresado por el usuario en unidades de Kilómetros.
Este plugin tambien permite trabajar con las geometrías de los recursos. En este caso, utilizando GEOTOOLS, se analiza el área de cada par de recursos entrantes, comparando si tienen un valor aproximado. Tomando en cuenta la tolerancia dada por el usuario, en Kilómetros cuadrados, se obtiene relaciones sameAs que indican la posibilidad de que se trate del mismo recurso del mundo real
En este plugin de igual manera que los anteriores tambien se trabaja con la información geométrica de cada recurso. En este caso, utilizando GEOTOOLS, se obtiene el perímetro de cada par de recursos entrantes y se los compara para verificar si tienen un valor aproximando. Consideranto el valor de tolerancia ingresado por el usuario en Kilómetros, se obtiene relaciones sameAs que indican la posibilidad de que se trata del mismo recurso del mundo real.
