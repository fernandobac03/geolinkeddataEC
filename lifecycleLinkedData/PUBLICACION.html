Una vez generado los RDF se procede a cargar estos archivos a un Triple Store, el mismo que debe cumplir la condición de soporte del estándar GEOSPARQL y también debe ofrecer el servicio SPARQL Endpoint para que el usuario pueda realizar las respectivas consultas.

Seleccionando el TripleStore

Existen TripleStore que actualmente se utilizan para la publicacion de los archivos RDF, entre ellos: Virtuoso, Fuseki, UseekM, etc., pero no todos los servidores presentan similares caracteristcias.Al trabajar con el modelo estandar GeoSPARQL se necesita de un TripleStore que soporte este estandar. Luego de un análisis de cada uno de los servidores posibles a utilizar se determino que Parliament es el unico servidor que actualmente soporta el mencionado estándar para la publicación y aplicación de funciones GEOSPARQL para la consulta de información.
